# Видение продукта

## Миссия

Euterpe — локальное веб-приложение для **управления и актуализации** персональной музыкальной библиотеки на домашнем сервере. Пользователь владеет файлами на диске; приложение помогает синхронизировать метаданные и каталог с облачными сервисами (на первом этапе — Qobuz).

## Целевая аудитория

- Домашний NAS / homelab
- Один пользователь или семья в LAN
- Активная подписка Qobuz (Studio / Sublime и т.п.)
- Нет требований к публичному SEO или multi-tenant SaaS

## Ключевые ценности

1. **Файлы — источник правды** — FLAC/MP3 на диске; БД пересобирается rescan.
2. **Приватность** — по умолчанию только localhost/LAN; без обязательного облака.
3. **Предсказуемость** — TDD, явная документация API, Docker с понятными volume.
4. **Музыка в центре** — UX и название отсылают к коллекции и качеству звука.

## Этап 1 (MVP интеграции)

- Авторизация Qobuz
- Синхронизация избранного (albums; опционально tracks/artists)
- Добавление/удаление избранного на Qobuz
- Получение URL для скачивания треков (качество FLAC/Hi-Res)
- Очередь загрузок (серверная; UI — позже)
- Docker-образ

## Вне scope этапа 1

- Редактирование тегов и обложек (Phase 5)
- Plex/Jellyfin/Navidrome
- Другие стриминги (Tidal, Deezer)
- Публичный доступ из интернета

## Нефункциональные требования

| Требование | Цель |
|------------|------|
| Деплой | Один `docker compose up` |
| БД | SQLite WAL, один writer |
| Бэкап | Копия `library.db` + каталог `/music` |
| Тестирование | Строгий TDD на всех фазах |

## Успех этапа 1

- Crate `euterpe-qobuz` покрыт тестами (mock + optional live)
- Документация `docs/05-qobuz` и `docs/06-library-euterpe-qobuz` соответствует коду
- Избранное Qobuz читается и изменяется из Rust CLI или integration test
